# מדריך עריכה — אתר אריאל חסן אדריכלות

מדריך קצר לעריכת התוכן, התמונות וההעלאה לאוויר. אין צורך בידע תכנותי כדי לבצע את רוב הדברים כאן.

---

## 1. הרצת האתר במחשב (לבדיקה לפני שינויים)

> ⚠️ חשוב: אל תריצו את האתר ישירות מתיקיית Google Drive — זה תוקע את השרת.
> העתיקו את הפרויקט לתיקייה מקומית במחשב (למשל `~/ariel-architecture`) והריצו משם.

```bash
cd ~/ariel-architecture
npm install      # פעם ראשונה בלבד
npm run dev      # מריץ את האתר בכתובת http://localhost:5173
```

---

## 2. החלפת תמונת הרקע הראשית (Hero)

תמונת הרקע הגדולה בראש העמוד נמצאת ב:

```
assets/Hero.png
```

פשוט החליפו את הקובץ הזה בהדמיה שלכם (אותו שם בדיוק: `Hero.png`).
**מומלץ:** תמונה רוחבית, ברזולוציה גבוהה (לפחות 2000px רוחב), JPG או PNG.

---

## 3. הוספה / החלפה של תמונות פרויקטים

יש שתי דרכים. בחרו אחת:

### דרך א' — דרך לוח הניהול (הכי קל, אבל דורש חיבור Supabase פעיל — ראו סעיף 6)
1. גשו לכתובת האתר עם `#admin` בסוף (למשל `http://localhost:5173/#admin`).
   *(אפשר גם ללחוץ פעמיים על שורת הזכויות בתחתית האתר.)*
2. התחברו והוסיפו/ערכו פרויקטים כולל העלאת תמונות.

### דרך ב' — קובץ הנתונים (זו הדרך שבה האתר בנוי כרגע)
כל הפרויקטים מוגדרים בקובץ אחד: **`data/projects.ts`**.
התמונות עצמן נמצאות ב-`public/projects/<שם-פרויקט>/` (הקובץ `01.jpg` הוא תמונת השער).

**להוסיף פרויקט חדש:**
1. צרו תיקייה: `public/projects/my-new-project/` ושימו בה את ההדמיות בשם `01.jpg`, `02.jpg`, ...
   (`01.jpg` = תמונת השער שמופיעה בכרטיס).
2. ב-`data/projects.ts`, העתיקו בלוק פרויקט קיים והדביקו בסוף הרשימה `initialProjects`, ועדכנו:
   ```ts
   {
     category: "architecture",            // architecture | interior | carpentry
     titleEn: "My New House", titleHe: "הבית החדש שלי",
     categoryEn: "Architecture", categoryHe: "אדריכלות",
     descriptionEn: "...", descriptionHe: "...",
     detailsEn: "...", detailsHe: "...",
     imageUrl: "/projects/my-new-project/01.jpg",
     images: ["/projects/my-new-project/01.jpg", "/projects/my-new-project/02.jpg"],
   }
   ```
3. שמרו — האתר יתעדכן אוטומטית.

**לשנות סדר פרויקטים:** הסדר ברשימה `initialProjects` = סדר ההצגה. הזיזו בלוקים למעלה/למטה.
**להחליף תמונת שער של קטגוריה:** ערכו את `imageUrl` במערך `categories` שבראש אותו קובץ.

## 4. עריכת טקסטים (עברית + אנגלית)

כל הטקסטים באתר מנוהלים בקובץ אחד:

```
contexts/LanguageContext.tsx
```

- הבלוק `en:` — כל הטקסטים באנגלית.
- הבלוק `he:` — כל הטקסטים בעברית.
- כל שורה היא `'מפתח': 'הטקסט'`. ערכו רק את הצד הימני (בין הגרשיים).
  שנו תמיד את שתי השפות כדי שיישארו תואמות.

פרטי הקשר (אימייל, טלפון) נמצאים גם בראש הקובץ `components/Contact.tsx`
(המשתנים `EMAIL`, `PHONE_DISPLAY`, `PHONE_INTL`) — עדכנו שם אם הם משתנים.
קישורי הרשתות החברתיות נמצאים ב-`components/Footer.tsx` (`socials`).

## 5. טופס יצירת הקשר

הטופס באתר שולח כרגע דרך תוכנת המייל של המבקר (`mailto`). זה עובד מיד וללא הגדרות.
אם תרצו שההודעות יישלחו אוטומטית למייל שלכם בלי לפתוח תוכנת מייל —
אפשר לחבר שירות חינמי כמו **Formspree** או **Resend** (דורש שינוי קטן ב-`components/Contact.tsx`).

## 6. הערה על Supabase (מסד הנתונים של הפרויקטים)

חיבור ה-Supabase הקיים בקובץ `.env` **כבר לא פעיל** (הכתובת לא נמצאת).
לכן האתר נופל אוטומטית לרשימת הפרויקטים שבקוד (`initialProjects`) — וזה תקין לאתר תדמית.

אם תרצו ניהול עצמי דרך לוח הבקרה (סעיף 3, דרך א'):
1. פתחו פרויקט חדש ב-supabase.com (חינם).
2. הריצו את `seed.sql` כדי ליצור את טבלת הפרויקטים.
3. עדכנו ב-`.env` את `VITE_SUPABASE_URL` ו-`VITE_SUPABASE_ANON_KEY`.

אחרת — פשוט התעלמו מ-Supabase והשתמשו בדרך ב' (קבצים מקומיים).

## 7. טיפים להכנת הדמיות שייראו מקצועי

- **אחידות:** שמרו על יחס גובה-רוחב דומה לכל ההדמיות (למשל 4:3 או 3:2) — זה נותן מראה נקי ומסודר.
- **רזולוציה:** לפחות 1600px ברוחב. לא צריך יותר מ-2400px (כדי שהאתר ייטען מהר).
- **פורמט:** JPG באיכות ~80% נותן איזון מצוין בין איכות למשקל. אפשר להמיר ב-squoosh.app.
- **תאורה ועקביות:** אם ההדמיות מגיעות ממקורות שונים — כדאי "ליישר" אותן (אותו טון/חום) בכלי עריכה
  או דרך Gemini/Photoshop, כדי שהגלריה תיראה כמו סדרה אחת.
- **משקל הקובץ הראשי (Hero):** הקובץ הנוכחי כ-2.7MB. מומלץ לדחוס ל-מתחת ל-500KB לטעינה מהירה.

## 8. העלאה לאוויר (Vercel — חינם)

1. צרו חשבון ב-vercel.com (אפשר עם Google).
2. העלו את תיקיית הפרויקט (או חברו ל-GitHub).
3. Vercel יזהה אוטומטית שזה פרויקט Vite. הגדרות:
   - Build Command: `npm run build`
   - Output Directory: `dist`
4. הוסיפו ב-Vercel את משתני הסביבה מתוך `.env` (אם משתמשים ב-Supabase).
5. לאחר ההעלאה אפשר לחבר את הדומיין `arielarch.co.il`.
